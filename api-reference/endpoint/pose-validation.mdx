---
title: 'Pose Validation'
openapi: 'POST /api/v1/analyzers/validate-pose/'
---

This endpoint validates the pose of a person in an image to ensure it meets the requirements for the clothing change operation. It supports both base64 encoded images and image URLs.

### Request Body (JSON):

Provide either `image_base64` or `image_url`, but not both.

<ParamField body="image_base64" type="string">
  Base64 encoded string of the image to validate. Can include data URI prefix (e.g., "data:image/jpeg;base64,...") which will be automatically stripped.
</ParamField>

<ParamField body="image_url" type="string">
  URL of the image to validate. The image will be downloaded and processed.
</ParamField>

<CodeGroup>
```json Request with Base64
  {
    "image_base64": "<base64_encoded_image>"
  }
```

```json Request with URL
  {
    "image_url": "https://example.com/image.jpg"
  }
```
</CodeGroup>

### Response (Success):

If the pose is valid, the response will be:

<CodeGroup>
```json Response
  {
    "correct": true
  }
```
</CodeGroup>

### Response (Failure):

If the pose is not valid, the response will be:

<CodeGroup>
```json Response
  {
    "correct": false,
    "message": "<error_message>"
  }
```
</CodeGroup>

### Possible Error Messages:

*   Serializer validation errors (e.g., missing required fields).
*   "Failed to download image from URL: {e}"
*   "Invalid image data: ..."
*   "Image decoding failed."
*   "Failed to load object detection model."
*   "Failed to load pose landmarker model."
*   "No person detected."
*   "More than one person detected."
*   "Could not detect pose landmarks."
*   "Nose not clearly visible."
*   "At least one wrist must be clearly visible."
*   "At least one foot must be clearly visible."
